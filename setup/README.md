## Setup

Open your Terminal application and navigate to your <code class="filepath">~/code/ga/lectures</code> directory:

```bash
cd ~/code/ga/lectures
```

Make a new directory called <code class="filepath">intermediate-python-for-scripting</code>, then enter this directory:

```bash
mkdir intermediate-python-for-scripting
cd intermediate-python-for-scripting
```

Then create an <code class="filepath">intermediate_python.py</code> file with:

```bash
touch intermediate_python.py
```

A quick note on convention - you'll note that we're using underscores (`_`) in this filename instead of dashes (`-`). This follows the [PEP style guide](https://peps.python.org/pep-0008/#package-and-module-names) for Python modules and packages.

Following this style guide is not required for our code to run, but following it means that you have a consistent style across any Python code you write. It also means that you have fewer decisions to make about how to best structure your code.

Open the contents of the directory in VS Code:

```bash
code .
```
